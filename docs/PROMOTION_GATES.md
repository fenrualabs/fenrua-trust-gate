# Promotion Gates

Status: R1 boundary record

## Current State

The repository is at R1: specification foundation. Source code and local tests
do not constitute a release, preview, service, assurance, or product
availability change.

## R1 to Later Implementation

Before an R2/R3 implementation claim, establish frozen normative schemas,
negative vectors, constrained policy semantics, evidence/receipt contract,
profile-specific signature review, replay design, threat-model updates, and
independently reviewable tests. The current crate boundary is preparation for
that work, not proof it is complete.

## R3 to R4 Limited Preview

The programme's governing gate requires public source, a real CLI, stable v1
schemas, signed release, SBOM, provenance, threat model, negative corpus,
benchmark, known limitations, clean reproduction, and owner approval. This R1
repository satisfies none of the release artifacts or operational evidence by
itself.

## Outstanding External Decisions

- owner-designated CODEOWNERS team and branch protection;
- security reporting endpoint and incident escalation;
- release signing custody and protected environment;
- approved SBOM/provenance/release process;
- supported-platform test evidence;
- private evidence vault and independent review process.
