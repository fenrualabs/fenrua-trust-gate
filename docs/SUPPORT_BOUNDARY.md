# Support Boundary

Status: no operational support service established

R2 source is an unreleased local prototype. It makes no availability,
response-time, uptime, compatibility, managed-support, hosted-operation, or
customer-data commitment. The CLI's `doctor` output is a static source
description, not a health check for a deployed system.

The source-level vulnerability-management policy is defined in
`VULNERABILITY_MANAGEMENT.md`, but it does not configure an operational security
or support function. Support can be defined only after named maintainers,
supported platforms, release process, private reporting and case-handling
operation, compatibility policy, and owner approval exist.
