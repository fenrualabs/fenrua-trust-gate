# Support Boundary

Status: no operational support service established

R1 source is provided only as an unreleased implementation foundation. No
availability, response-time, uptime, compatibility, managed support, hosted
operation, or customer-data handling commitment is made. The CLI's `doctor`
output is a static self-description, not a diagnostic of a deployed system.

Any future support promise requires named maintainers, supported platforms,
release process, vulnerability process, compatibility policy, and owner
approval.
